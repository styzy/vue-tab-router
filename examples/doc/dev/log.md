# 更新日志

:::success

当前稳定版本： `0.9.8`

:::

## 版本： `0.9.8`

* 更新时间： `2024-04-16`

* 更新内容：
  + 修复`open()`方法当省略`query`参数时，无法触发路由视图参数更新的bug。

## 版本： `0.9.6`

* 更新时间： `2023-12-28`

* 更新内容：
  + 重构视图组件生命周期底层监听机制以修复组件复用导致关闭异常的bug。

## 版本： `0.9.5`

* 更新时间： `2023-12-27`

* 更新内容：
  + 修复`TabRoute`路由对象工具方法`$updateLocation()`调用导致标签页无法关闭的bug。

## 版本： `0.9.4`

* 更新时间： `2023-12-27`

* 更新内容：
  + `TabRoute`路由对象工具方法优化更新。

## 版本： `0.9.3`

* 更新时间： `2023-12-26`

* 更新内容：
  + `TabRoute`路由对象现在不仅在渲染视图的直接子组件内可访问，还在所有的后代子组件内可访问。

## 版本： `0.9.2`

* 更新时间： `2023-12-15`

* 更新内容：
  + 优化导航的参数更新行为，现在`open()`、`focus()`、`reload()`方法当指定`query`参数时，会实时更新参数。

## 版本： `0.9.1`

* 更新时间： `2023-12-14`

* 更新内容：
  + 修复全局后置钩子函数中的`focus`类型无法被正确触发的bug。

## 版本： `0.9.0`

* 更新时间： `2023-12-13`

* 更新内容：
  + `TabRouter`实例新增响应式属性`currentRoutes`用于获取当前聚焦的路由对象。
  + `TabRouter`实例新增`getActiveRoutes()`方法用于获取已激活的路由对象。
  + `TabRouter`实例部分方法包括`closeAll`、`getRoutes`、`getActiveRoutes`，支持通过指定命名视图的名称（`router`）来实现有针对性的操作。

## 版本： `0.8.4`

* 更新时间： `2023-12-08`

* 更新内容：
  + 修复标题组件在关闭非聚焦的路由视图后，会失去聚焦路由视图的bug。

## 版本： `0.8.3`

* 更新时间： `2023-12-08`

* 更新内容：
  + 修复标题组件`<TabRouterLink>`无法正确提供作用域插槽参数`isVisited`的bug。

## 版本： `0.8.2`

* 更新时间： `2023-12-08`

* 更新内容：
  + 修复关闭所有路由后，数据无法同步更新相应的bug。

## 版本： `0.8.1`

* 更新时间： `2023-12-08`

* 更新内容：
  + 优化导航守卫，现在通过`focus()`方法聚焦已经处于聚焦状态的路由时，可以成功触发`全局前置导航守卫（beforeEach）`的`聚焦类型（NAVIGATE_TYPES.FOCUS）`了，但因为此聚焦行为属于无意义的聚焦行为，为了避免不必要的的性能消耗，因此`全局后置钩子`将`不会`被触发。

## 版本： `0.8.0`

* 更新时间： `2023-12-07`

* 更新内容：
  + 新增命名视图功能，可支持同时独立渲染多个标签页视图。
  + 优化框架性能。
